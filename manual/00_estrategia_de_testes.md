# Estratégia de Testes – Avaliação de Qualidade

## Objetivo

Avaliar a qualidade de duas páginas disponibilizadas para análise:

- https://qualidade.apprbs.com.br/certificacao
- https://qualidade.apprbs.com.br/site

A avaliação considera aspectos funcionais, visuais e de experiência do usuário.

---

## Escopo

### Em escopo

- Testes exploratórios
- Testes manuais funcionais
- Avaliação de interface e layout (UI)
- Verificação de hierarquia visual e consistência
- Navegação e funcionamento de CTAs
- Validação de formulários
- Responsividade básica (mobile, tablet, desktop)
- Testes automatizados de smoke

---

### Fora de escopo

- Testes de carga ou performance avançada
- Auditoria completa de acessibilidade
- Testes de segurança
- Integrações internas do sistema

---

## Tipos de testes aplicados

### Testes exploratórios

Realizados para identificar problemas visuais, inconsistências e comportamentos inesperados.

### Testes manuais

Casos de teste definidos para validar fluxos principais da aplicação.

### Testes automatizados

Testes automatizados simples para validar:

- carregamento das páginas
- disponibilidade (HTTP)
- presença de elementos principais

---

## Critérios de qualidade avaliados

- Clareza de hierarquia visual
- Consistência de interface
- Legibilidade
- Organização do layout
- Clareza de CTAs
- Ausência de conteúdo placeholder
- Experiência geral do usuário
